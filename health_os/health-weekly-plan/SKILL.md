# Health Weekly Plan（周健康计划）

## 触发

- **定时**：由调度系统每周一 07:00 调用
- **手动**：用户说「排周健康计划」「本周运动计划」

## 计划 vs 记录 —— 严格区分

| 载体 | 存放内容 | 写入时机 |
|------|---------|---------|
| **飞书 Calendar** | 具体训练安排（时间、动作、组数） | 周计划生成时 |
| **Base `训练主题`** | 当日训练主题（如「上肢力量」） | 周计划生成时 |
| **Base `训练强度`** | 计划强度（轻度/正常/加码） | 周计划生成时 |
| **Base `运动_1/2/3`** | ⚠️ **实际完成的记录** | **只在晚间日结或用户手动记录时写入** |

> 🚫 周计划绝不写 `运动_1/2/3`、`运动类型`、`运动时长`、`运动完成率`。

## 金奇作息画像

| 指标 | 值 |
|------|-----|
| 起床中位数 | **08:30**（周五六约 09:00） |
| 主训练锚点 | **19:30**（历史主力，不可动） |
| 下午微训 | 15:00（有历史验证，出勤率偏低） |
| 上午 | **0 次历史记录，不排** |

## 训练主题模板（7 天循环）

| 星期 | 主题 |
|------|------|
| 周一 | 上肢力量 |
| 周二 | 有氧耐力 |
| 周三 | 下肢力量 |
| 周四 | 上肢力量 |
| 周五 | 有氧耐力 |
| 周六 | 全身综合 |
| 周日 | 恢复日 |

## 每日结构

| 场次 | 时段 | 时长 |
|------|------|------|
| 晚间主训 | **19:30**（周日 21:00） | 30–45min |
| 下午微训 | 15:00（周六 15:30，周日 16:00） | 10–15min（本周酌情安排） |

## 工作流程

```
1. 读取操作指引 + 脚本分析
   references/config.md
   python3 health-system/scripts/health_analytics.py --weekly-plan

2. 必读
   content/profile.md（长期画像：作息锚点、护膝等）
   content/{YYYY}/{MM}/monthly-plan.md（本月运动/饮食目标）
   references/safety-rules.md

3. 安全预检 → 确定本周强度

4. 【强制】先删除本周已有日历事件

   Step 1：查询本周已有事件
   lark-cli calendar +agenda \
     --start MONDAY_DATE --end SUNDAY_DATE+1 \
     --as user --format json > /tmp/week_events.json

   Step 2：逐个删除
   lark-cli calendar events delete \
     --calendar-id $CALENDAR_ID \
     --event-id $EVENT_ID \
     --as user

   Step 3：验证已清空（确认事件数=0 后，才执行 Step 4）

   Step 4：创建新事件（每天晚间+可选微训）

5. 写入 Base（仅计划意图）
   upsert_by_date('YYYY-MM-DD', {'训练主题': '...', '训练强度': '正常'})
   ⚠️ 不写运动_1/2/3

6. 写入本地 state/current-week.json
   记录本周每天的训练主题/强度/时段（供 daily-remind 读取）
```

## current-week.json 格式

```json
{
  "week": "2026-W27",
  "generated_at": "2026-07-01T07:00:00+08:00",
  "days": {
    "2026-07-01": {"theme": "上肢力量", "intensity": "正常", "slots": ["19:30"]},
    "2026-07-02": {"theme": "有氧耐力", "intensity": "正常", "slots": ["19:30"]},
    "2026-07-06": {"theme": "恢复日",   "intensity": "轻度",  "slots": ["21:00"]}
  }
}
```

## 调整规则

| 条件 | 调整 |
|------|------|
| 上周完成率 < 50% | 晚间主训降时/降组数 |
| 安全标记有异常 | 周一强制恢复 |
| 体重单周升 > 0.5kg | 加强饮食提示 |

## 防重复检查点

- [ ] 查询本周已有事件
- [ ] 删除所有已有事件
- [ ] 验证本周事件数 = 0
- [ ] 创建新事件
- [ ] 验证创建成功
- [ ] 写入 state/current-week.json

## 故障排查

若创建后发现重复事件：
1. 立即停止创建
2. 查询本周所有事件
3. 删除重复事件（保留最早的）
4. 检查删除逻辑是否失败
5. 向用户报告异常

## 配置引用

`references/config.md`。周计划须符合当月 `monthly-plan.md` 中的运动频率与类别目标。
