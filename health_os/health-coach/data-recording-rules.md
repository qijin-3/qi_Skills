# 数据记录规则（Data Recording Rules）

## 核心原则

**每天只有一条记录**：每个日期在 Base 中只能有一条记录。

---

## 睡眠记录规则

### 睡眠时长归属

**睡眠时长算在上床那天**（不是起床那天）。

**示例**：
- 用户 6月25日凌晨1点睡觉
- 用户 6月25日早上7点起床
- **睡眠时长算在6月24日**（上床那天）

**6月24日记录**：
- 就寝时间：2026-06-25 01:00
- 起床时间：2026-06-25 07:00
- 睡眠时长：6 小时

**6月25日记录**：
- 就寝/起床 均为 null（这次睡眠已算在 6月24日）

## 卡路里估算规则（必填，不可跳过）

**记录任何餐次时，必须同步写入对应 `_卡路里` 字段。**

### 估算精度

| 场景 | 精度要求 | 备注字段 |
|------|---------|---------|
| 自己做饭，食材明确 | ±50 kcal | 无需备注 |
| 外卖/外食，菜品已知 | ±100 kcal | 饮食备注写「外食」 |
| 外卖/外食，菜品模糊 | 给区间中值 | 饮食备注写「外食估算」 |
| 聚餐，无法准确估算 | 给保守高估值 | 饮食备注写「聚餐估算」 |

**禁止**：以「外食不清楚」为由留空。实在不确定也要填一个保守估算值。

### 常见食物卡路里参考表

| 食物 | 份量 | 参考 kcal |
|------|------|-----------|
| 鸡蛋 | 1个 | 75 |
| 无糖豆浆 | 300ml | 50 |
| 全脂牛奶 | 250ml | 155 |
| 燕麦片（干） | 50g | 185 |
| 糙米饭 | 150g | 210 |
| 白米饭 | 150g | 195 |
| 全麦面包 | 1片（30g） | 75 |
| 包子 | 1个 | 200 |
| 鸡胸肉（清蒸） | 100g | 110 |
| 猪肉（红烧） | 100g | 260 |
| 鱼（清蒸） | 100g | 120 |
| 虾 | 100g | 90 |
| 西兰花 | 200g | 68 |
| 黄瓜 | 200g | 30 |
| 豆腐 | 100g | 80 |
| 普通外卖便当 | 1份 | 700 |
| 家常炒菜套餐 | 1份 | 600 |
| 日式定食 | 1份 | 550 |

### evening-close 补填逻辑

晚间日结时，若发现当日有餐次文本但对应卡路里为空，自动补估算并写入：

```python
for meal in ['早餐', '午餐', '晚餐', '加餐']:
    if record.get(meal) and not record.get(f'{meal}_卡路里'):
        estimated = estimate_calories_from_text(record[meal])
        upsert_by_date(date, {f'{meal}_卡路里': estimated})
```

## 创建记录流程

### 强制检查

创建前必须检查当天是否已有记录：
1. 有 → 获取 record_id，执行更新
2. 没有 → record_id = None，执行创建

### 检查命令（按日期查找）

```python
from health_base import find_record_by_date
record = find_record_by_date('2026-07-01')
# record 为 None → 创建；否则 → 更新
```

## Calendar 时段规则

**每个时段最多一个运动事件**：

| 时段 | 时间范围 |
|------|---------|
| 早上 | 06:00–12:00 |
| 下午 | 12:00–18:00 |
| 晚上 | 18:00–23:59 |

**新增或推迟运动事件前，必须先查目标时段**：
```bash
lark-cli calendar +agenda \
  --start YYYY-MM-DD --end YYYY-MM-DD+1 \
  --as user --format json
```

若目标时段已有运动事件：告知用户「该时段已有 [事件名]，是替换还是合并？」，等用户确认再操作。

**Calendar 操作规则**：
- 删除事件：必须用户明确说「删除/取消/不练了」
- 新增事件：必须用户明确说「加一场/增加运动/补个训练」
- 调整内容/时间：情境变更时 update 已有事件（不删除再创建）
