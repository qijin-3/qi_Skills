# Health Daily Remind（每日健康提醒）

## 触发

- **定时**：由调度系统每天 07:30 调用
- **手动**：用户说「今日健康计划」「健康提醒」

## ⚠️ 只读！不做任何写入

不写入 Base、不修改 Calendar、不创建任何文件。

## ⚠️ Calendar 是计划的唯一来源

**绝对禁止**：重新生成或修改今日训练计划。

必须执行：
1. 读取飞书 Calendar 中今日的训练事件
2. 将 Calendar 中的训练内容直接呈现给用户
3. 不添加、不修改、不重新生成任何训练计划

## 工作流程

```
1. 读取操作指引
   ~/SynologyDrive/Sync/OS/Health_OS/references/config.md

2. 读取本地上下文
   content/profile.md（长期习惯，非本月目标）
   content/{YYYY}/{MM}/monthly-plan.md（本月饮食/运动侧重点）
   state/current-week.json

3. 读取 Base 上下文（近期数据摘要）
   python3 health-system/scripts/health_analytics.py --daily-remind

4. 【强制】读取飞书 Calendar 今日训练事件
   lark-cli calendar +agenda \
     --start YYYY-MM-DD --end YYYY-MM-DD+1 \
     --as user --format json
   提取 15:00 和 19:30 的训练内容

5. 生成前情摘要 + 今日时间线（使用 Calendar 实际内容）

6. 飞书发送（≤10 行）
```

## 金奇作息速查

| 指标 | 值 |
|------|-----|
| 起床中位数 | 08:30（周五六 ~09:00） |
| 主训练锚点 | **19:30**（历史主力时段） |
| 下午微训 | 15:00（计划日才有） |
| 上午训练 | **0 次历史记录 → 不提醒上午运动** |

## 消息格式（≤10 行）

### 有下午微训的工作日

```
📋 周四 · 上肢力量

昨天没运动，今天晚间稳扎稳打。

15:00 🧘 微训 → 靠墙静蹲 2×30s，腿髋拉伸
19:30 💪 上肢 → 跳绳 300，俯卧撑 4×15，仰卧起坐 4×20

🍳 早餐：水煮蛋2个+无糖豆浆1杯+全麦面包1片
🥗 午餐：糙米饭150g+清蒸鱼200g+西兰花200g
🍚 晚餐：小米粥1碗+凉拌黄瓜+鸡胸肉150g

有变化直接说。
```

### 仅晚间

```
📋 周三 · 下肢力量

护膝：膝盖不超脚尖、不内扣。

19:30 🦵 下肢 → 自重深蹲 4×15，臀桥 3×15

🍳 早餐：牛奶1杯+燕麦片50g+香蕉1根
🥗 午餐：杂粮饭100g+虾仁150g+青菜200g
🍚 晚餐：蒸蛋羹+紫菜蛋花汤+清炒豆角
```

### 恢复日

```
📋 周日 · 恢复日

好好休息。

16:00 🧘 散步+泡沫轴
🍳 全天轻食
```

## 前情摘要规则

| 场景 | 怎么说 |
|------|--------|
| 连续完成 | 「连续 N 天完成，保持节奏。」 |
| 昨日未运动 | 「昨天休息了，今天晚间完成。」 |
| 体重上升 | 「体重连涨，今天晚餐控碳水。」 |
| 饮食偏差 | 「昨天高升糖偏多，今天早餐换蛋白质。」 |

## 饮食建议原则

- ✅ 每餐具体食物（水煮蛋2个，不是「高蛋白」）
- ✅ 包含份量（牛奶1杯，糙米饭150g）
- ✅ 符合减脂目标
- ❌ 标签式建议（「蛋白+蔬菜」→ 换成「水煮蛋2个+西兰花200g」）

## 硬规则

- 不提醒上午运动
- 不写 Base，不改 Calendar
- 具体时间锚点：15:00 / 19:30
- ≤10 行，先结论再论据

## 配置引用

`references/config.md`（操作指引）。Token/字段 ID 由脚本读取 `system.json`，AI 不碰。
