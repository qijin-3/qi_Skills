# Mode A：北极星初始化

> 适用于：系统首次运行、`north-star.md` 不存在或用户明确要求从零建立北极星。

## 前置说明（发给用户）

```
我们要建立你的北极星——整个系统的最高约束层。
北极星一旦建立，全年默认不改，所有月目标、周主线都会围绕它来设计。

所以我会先问你几个问题，帮你想清楚方向，而不是直接让你填模板。
大概需要 10–15 分钟，准备好了吗？
```

等用户确认。

---

## Step 1：方向校准问题（必须全部问，不可跳过）

逐一提问，等用户逐条回答：

```
Q1. 你现在主要的精力放在什么上？这件事 3 年后还值得继续吗？

Q2. 过去半年，哪件事让你感到最强的心流或成就感？

Q3. 如果你只能做一件事做到死，你会选什么？

Q4. 你最不想成为什么样的人？（反向定义，有时候更清晰）
```

记录用户的完整回答，不要概括或简化，原话更有价值。

---

## Step 2：结合 About Me 起草北极星

Read `Personal-os/content/about-me.md`，结合用户四个问题的回答，起草北极星草案：

```
【北极星草案】

核心方向（1-2 句话）：
{提炼用户回答的方向，与 About Me 中的身份/目标保持一致}

三个战略支柱：
┌─────────────────────────────────────────────────────────────┐
│ 支柱 1：{名称}                                               │
│ 说明：{一句话，说这个支柱具体做什么}                          │
│ 对应清单：{Writing / Projects / Growing}                    │
├─────────────────────────────────────────────────────────────┤
│ 支柱 2：{名称}                                               │
│ 说明：{一句话}                                               │
│ 对应清单：{...}                                             │
├─────────────────────────────────────────────────────────────┤
│ 支柱 3：{名称}（可选，若方向足够聚焦可只用 2 个）             │
│ 说明：{一句话}                                               │
│ 对应清单：{...}                                             │
└─────────────────────────────────────────────────────────────┘

全年刻意放弃：
- {一个放弃项}
- {一个放弃项}
- {建议至少 3 条，足够有杀伤力}

年底衡量标准（30 秒内能判断是否达到）：
- 支柱 1：{具体数字/事件}
- 支柱 2：{具体数字/事件}
- 支柱 3：{具体数字/事件}

你觉得这个方向准确吗？有什么想调整的？
```

---

## Step 3：质量门（必须全部通过才写入）

用户回应后，对草案做四关检查，输出诊断报告：

**① 聚焦检查**
- 三支柱加起来，一个人全力以赴能做到吗？
- 有没有两个支柱在时间/精力上明显冲突？
- ❌ 发现冲突 → 建议合并或取舍，给 2 个方案

**② 可测量检查**
- 年底衡量标准，能在 30 秒内判断达到与否？
- ❌ 模糊 → "「{标准}」月底怎么判断？给一个具体的数字或事件"

**③ 杠杆检查**
- 三支柱中，至少有一个属于媒体杠杆（写作/内容输出）或产品杠杆（独立产品）？
- ❌ 全是能力积累 → "当前草案缺少外部可验证的产出支柱，建议加一个写作或产品方向"

**④ 放弃检查**
- 刻意放弃清单，是否足够有杀伤力？能对抗常见诱惑？
- ❌ 太泛（"不做低价值事情"）→ "这条太模糊，举个具体的诱惑，我来帮你写清楚"

所有关都通过（或用户打磨后通过）→ 进入 Step 4。

---

## Step 4：最终确认

```
以下是最终版本：

{草案完整内容}

确认写入吗？写入后将作为本年度全局约束。
（说"确认"或"OK"或"好的"即可）
```

**必须等待明确确认**，用户说"先看看"/"再想想"→ 保存草案到 `Personal-os/state/north_star_draft.json`，告知用户草案已保存。

---

## Step 5：写入文件与目标表

用户确认后：

**5a. 写入 `Personal-os/content/north-star.md`**

按以下模板结构写入（不要改变标题层级，其他技能依赖此结构）：

```markdown
# 北极星

> 最后更新：{YYYY} 年度（初始化）
> 本文档是系统的最高约束层。月目标、周主线的任何变更都应回溯到此处检验合理性。
> AI 守护规则：未经「中途修订流程」，AI 不得主动建议修改本文档。

## 核心方向

{1-2 句话}

## 三个战略支柱

| 支柱 | 说明 | 对应清单 |
|------|------|----------|
| {支柱 1} | {说明} | {清单} |
| {支柱 2} | {说明} | {清单} |
| {支柱 3} | {说明} | {清单} |

## 刻意放弃（全年）

- {放弃项}
- {放弃项}

## 衡量标准（年底 30 秒内判断）

| 支柱 | 年底判断标准 |
|------|------------|
| {支柱 1} | {标准} |
| {支柱 2} | {标准} |
| {支柱 3} | {标准} |

## 已知风险 & 守护信号

> 系统在偏差扫描时，如果检测到以下信号，必须预警：

| 风险 | 触发信号 |
|------|---------|
| {风险描述} | {可检测信号} |

## 修订记录

| 日期 | 修订内容 | 触发原因 |
|------|----------|----------|
| {YYYY-MM-DD} | 初版制定 | 系统初始化 |
```

**5b. 在 `TABLE_NORTH_STAR` 写入三个支柱记录**（每个支柱一条，共三条）：

```bash
# 对三个支柱各执行一次，替换对应内容
lark-cli base +record-upsert --as user \
  --base-token "<FEISHU_BASE_TOKEN>" \
  --table-id "<TABLE_NORTH_STAR>" \
  --json '{
    "支柱名称":"<支柱名，如：内容杠杆>",
    "核心方向":"<该支柱的一句话说明>",
    "年底衡量标准":"<30秒内能判断的标准>",
    "对应清单":"<Writing/Projects/Growing>",
    "状态":"进行中",
    "年份":"<YYYY>"
  }'
```

三条记录写入后，记录各自的 `record_id`，供后续 `monthly-goal` 创建月度目标时填写关联字段使用。

**5c. 输出确认摘要**

```
✅ 北极星已建立！

核心方向：{核心方向}
三支柱：{支柱1} / {支柱2} / {支柱3}

文件：content/north-star.md
```
