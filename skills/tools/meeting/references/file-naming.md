# 文件命名与处理流程

## 录音文件（手工命名）

- 支持格式：`.m4a`、`.mp3`、`.wav`、`.aac`、`.ogg`、`.flac`、`.wma`
- 单个会议：`录音2026-01-08.m4a`
- 多个会议：`录音2026-01-09-1.m4a`、`录音2026-01-09-2.wav`
- 必须以 `录音` 开头，且文件名包含有效日期 `YYYY-MM-DD`

## 逐字稿文件（工具生成）

- 格式：`录音文件名_原文.md`
- 示例：`录音2026-01-09-1_原文.md`

## 会议纪要文件（AI 生成）

- 格式：`会议纪要YYYY-MM-DD-N.md`（序号与录音/逐字稿一致）
- 示例：`会议纪要2026-01-09-1.md`、`会议纪要2026-01-09-2.md`

## 处理流程示意

```
~/Downloads/                                   源文件夹
├── 录音2026-01-09-1.m4a                      原始录音
├── 录音2026-01-09-1_原文.md                   原始逐字稿
         ↓ 脚本处理
~/Downloads/2026-01-09/                        日期文件夹
├── 录音2026-01-09-1.m4a                      复制录音
├── 逐字稿2026-01-09-1.md                      重命名逐字稿
         ↓ 自动归档
<archive_dir>/2026-01-09/                      归档目录
├── 录音2026-01-09-1.m4a
├── 逐字稿2026-01-09-1.md
└── 会议纪要2026-01-09-1.md                    AI 生成
```

## 配对原则

录音、逐字稿、会议纪要三者必须使用相同序号，不能重新编号。

```
录音2026-01-07-1.m4a       ← 序号 1
逐字稿2026-01-07-1.md     ← 序号 1
会议纪要2026-01-07-1.md   ← 序号 1
```
