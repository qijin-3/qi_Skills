# 故障排除

## 脚本执行失败

```bash
python3 --version
ls -la scripts/process_meetings.py
chmod +x scripts/process_meetings.py
```

## 找不到录音文件

- 文件名必须以 `录音` 开头
- 扩展名必须是支持的音频格式
- 正确：`录音2026-01-09-1.m4a` ✅
- 错误：`recording-2026-01-09.m4a` ❌（缺少中文前缀）
- 错误：`录音2026-01-09-1.txt` ❌（不支持的格式）

## 找不到逐字稿

- 格式必须为 `录音文件名_原文.md`
- 正确：`录音2026-01-09-1_原文.md` ✅
- 错误：`逐字稿2026-01-09-1.md` ❌

## 无法提取日期

- 文件名须包含 `YYYY-MM-DD` 且日期有效
- 正确：`录音2026-01-09-1.m4a` ✅
- 错误：`录音2026-99-99.m4a` ❌（无效日期）

## 归档目录问题

- 脚本会自动创建 `archive_dir`
- 确保对归档位置有读写权限

## 文件已存在

- 默认跳过，不覆盖
- 需要覆盖时，将 `config.json` 中 `overwrite` 设为 `true`

## 性能参考

- 小批量（<10 个文件）：即时处理
- 中批量（10–50 个）：几秒到几十秒
- 大批量（>50 个）：建议分批或加 `--no-archive`
