# Product Namer

基于世界顶级命名机构 Lexicon Branding（代表作：BlackBerry, Pentium, Sonos, Azure, Vercel）的方法论，为您的产品生成专业的品牌名称。

## 快速开始

为产品命名的基本流程：

1. **收集产品信息**：了解产品是什么、目标用户、核心优势
2. **钻石框架分析**：定义"赢"、"有什么"、"缺什么"、"说什么"
3. **生成名称选项**：应用伪装简报法和声音符号学
4. **竞争度检查**：使用搜索工具检查名称可用性
5. **输出详细分析**：为每个名称提供声音符号学和竞争度分析

## 核心工作流程

### 第一步：理解产品并定义战略

在生成任何名称之前，先使用**钻石框架**理解产品：

1. **赢 (Win)**：市场地位愿景是什么？
2. **有什么 (Have)**：现有资产、技术优势、团队背景？
3. **缺什么 (Need)**：缺失的资源或短板？
4. **说什么 (Say)**：最关键 - 向市场传递什么体验？

**关键原则**：不要描述产品功能，而是定义品牌体验。例如：
- ❌ "一个快速的 AI 编程工具"
- ✅ "让编程像冲浪一样流畅自由"

### 第二步：判断语言需求

根据产品特征自动判断语言策略：

| 场景 | 语言策略 | 示例 |
|------|---------|------|
| 仅面向中国市场 | 仅中文 | 飞书、钉钉 |
| 国际化/全球市场 | 仅英文 | Vercel, Sonos |
| 中国市场为主，但需国际兼容 | 中文为主 + 英文辅助 | 淘宝 (Taobao) |
| 同时面向中外市场 | 双语方案 | 提供"中文名 + 英文名"配对 |

**判断依据**：
- 目标用户市场（中国/全球/特定地区）
- 产品类型（本地化服务 vs. 通用工具）
- 公司品牌定位（本土 vs. 国际）

### 第三步：生成名称选项

应用三种命名策略：

#### 1. 伪装简报法 (The Excursion Method)

即使只有一个产品，也从多个角度发散：

- **角度 A（直球）**：基于真实产品信息
- **角度 B（类比）**：假设为不同类别命名（运动、自然、音乐）
- **角度 C（跨界）**：从其他领域寻找灵感

#### 2. 声音符号学 (Sound Symbolism)

根据产品特性选择字母：

| 字母 | 特性 | 适用场景 |
|------|------|---------|
| **V** | 活力、速度 | 科技、创新、动态产品 |
| **B** | 可靠、稳固 | 金融、安全、基础设施 |
| **Z** | 科技感、未来 | AI、前沿技术 |
| **X** | 快速、创新 | 开发者工具、高性能产品 |

详细参考：[sound-symbolism.md](references/sound-symbolism.md)

#### 3. 行业模式匹配

根据产品行业选择命名风格：

- **科技/SaaS**：生造词、隐喻词（Vercel, Azure）
- **消费品牌**：情感词、具象词（Apple, Nike）
- **开发者工具**：技术词根、组合词（GitHub, Vercel）
- **金融科技**：信任词汇、稳重词（Square, Stripe）
- **AI/ML**：智能暗示、流动词（Neural, Flow）

详细参考：[naming-patterns.md](references/naming-patterns.md)

### 第四步：竞争度检查

为每个候选名称进行搜索竞争度分析：

```bash
# 检查单个或多个名称
python3 scripts/search_competition.py "名称1" "名称2" "名称3"

# 或从文件读取
python3 scripts/search_competition.py --file names.txt
```

**解读结果**：
- 🟢 **LOW**：竞争较少，更容易建立品牌
- 🟡 **MEDIUM**：有一定竞争，但仍有空间
- 🔴 **HIGH**：竞争激烈，需要强有力的差异化
- ⚪ **UNKNOWN**：需要手动检查 Google/百度

### 第五步：输出详细分析

为每个名称提供完整分析，格式如下：

```
## 名称：[Name]

### 基本信息
- **发音**：[国际音标或拼音]
- **词源/构造**：[词根组合或生造词逻辑]
- **含义/隐喻**：[传达的核心概念]

### 声音符号学分析
- **核心字母**：[V/B/Z/X 等]
- **情感基调**：[活力/可靠/科技/创新]
- **流畅性**：[易于发音和记忆的程度]

### 命名策略
- **钻石框架对接**：如何回应"说什么"
- **差异化优势**：与竞品的区别
- **品牌故事**：可展开的品牌叙事

### 竞争度检查
- **搜索结果数量**：[约 XX 条]
- **竞争程度**：[LOW/MEDIUM/HIGH]
- **域名可用性**：[.com/.ai/.io 状态]

### 适用场景
- **目标用户**：[精准定位]
- **行业匹配度**：[高/中/低]
- **国际化潜力**：[适合哪些市场]
```

## 输出原则

### 数量控制
- 初次生成：提供 10-15 个不同方向的选项
- 筛选后：聚焦 3-5 个最强选项进行详细分析

### 风格多样性
每个输出应包含混合风格：
- 2-3 个生造词（创新独特）
- 2-3 个隐喻词（联想丰富）
- 2-3 个组合词（1+1=3 效应）
- 1-2 个意外选项（跨界灵感）

### 避免的陷阱
- ❌ 描述性名称："Cloud Pro", "Fast Code"
- ❌ 技术术语堆砌：滥用 "Tech", "Soft", "Sys"
- ❌ 过于熟悉的组合：缺乏差异化
- ❌ 难以发音：破坏流畅性原则

### 追求的目标
- ✅ **两极分化**：一半人喜欢、一半人争议 = 有能量
- ✅ **不舒服感**：陌生但有吸引力 = 有独特性
- ✅ **流畅性**：易读易记 = 易传播
- ✅ **故事性**：能展开品牌叙事 = 长期资产

## 验证方法建议

在输出名称后，建议用户使用以下验证方法：

### 1. 原型测试
- 将名称放在 T 恤、新闻头条、产品截图上
- 观察在真实场景中的表现

### 2. 竞争对手测试
- 告诉团队："发现一个新竞争对手叫 [名称]"
- 观察是否有好奇和追问

### 3. 两极分化检查
- 在内部讨论，观察是否有激烈争论
- 争议代表能量，"还行"代表平庸

## 使用示例

### 示例 1：AI 编程工具命名

**用户输入**：
"帮我为一个 AI 编程工具命名，主要功能是实时代码补全和智能重构，目标用户是前端开发者。"

**技能响应**：
1. 使用钻石框架：赢=开发者效率提升，有=AI 技术，缺=品牌认知，说=编程如冲浪般流畅
2. 判断语言：国际化工具 → 英文为主
3. 生成选项：Windsurf, Flow, Velox, Novva, CodeWave 等
4. 声音符号学分析：V (活力) + Flow (流动)
5. 竞争度检查：使用脚本验证
6. 输出详细分析

### 示例 2：中国消费品牌命名

**用户输入**：
"我有一个新式茶饮品牌，主打健康和年轻化，需要中文名。"

**技能响应**：
1. 使用钻石框架：赢=年轻人首选，有=健康配方，缺=品牌知名度，说=健康潮流生活
2. 判断语言：仅中国市场 → 中文为主
3. 生成选项：使用叠词、开口音、美好寓意
4. 声音符号学分析：开口音 (显大气) + 上升调 (发展感)
5. 输出详细分析
