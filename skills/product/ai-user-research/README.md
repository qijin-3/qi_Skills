# AI 用户研究 - LLM 选择建议

并非所有 LLM 在用户研究分析方面都同样出色。基于在 Claude、ChatGPT 和 Gemini 之间进行的 100 多次比较测试。

## 快速推荐

**对于分析工作，如果您有选择，请使用 Claude。**

Claude 在保持基于实际数据的同时覆盖更多内容。您获得深度和广度，而无需过多的提示。权衡：它不会为您过滤，因此您需要验证主题是否有充分的证据支持。

## 详细比较

### Claude（推荐用于分析）

**最适合：** 具有深度和细微差别的彻底分析

**优势：**
- 提供更多引言，用更少的提示覆盖更多内容
- 保持基于实际数据
- 提供洞察的广度和深度
- 对置信度评分更为保守
- 发现验证的模式和半成型的假设

**劣势：**
- 不会为您过滤 - 验证的模式和半成型的想法平等呈现
- 您需要验证主题是否有充分的证据支持
- 可能会给您"整个大脑倾倒"

**何时使用：**
- 面试和调查分析的主要选择
- 当您想要全面覆盖时
- 当深度和细微差别比整洁的摘要更重要时
- 当您愿意验证证据质量时

**最佳实践：**
- 始终验证引言和证据质量
- 准备好审查更多输出
- 使用显式矛盾检查提示词
- 要求置信度水平以便稍后过滤

---

### Gemini（和 NotebookLM）

**最适合：** 高度证据化的主题和视频分析

**优势：**
- 提供更少的主题，但有更强的证据链接
- 独特优势：可以分析视频中的非语言行为（其他模型尚不能）
- 强有力的证据引用

**劣势：**
- 通常选择过短的客户声音片段
- 需要多次提示以获得完整性
- 在追求严格证据时可能会遗漏更广泛的主题

**何时使用：**
- 当您有视频面试并想分析非语言行为时
- 当您更喜欢高度证据化的主题而不是全面覆盖时
- 当您愿意多次提示以获得完整性时

**最佳实践：**
- 明确要求更长的引言
- 期望提示 2-3 次以确保完整性
- 适用于在别处发现的主题的最终验证

---

### ChatGPT

**最适合：** 最终框架设计和利益相关者沟通

**优势：**
- 三个模型中最有创意的
- 擅长为特定受众包装发现
- 适合将研究转化为利益相关者演示

**劣势：**
- **对于真实证据最不可靠** - 将引言组合成拼接
- 可能遗漏重要主题（例如，价格敏感性反应）
- "逐字引言"经常被总结或组合
- 最容易产生幻觉和弗兰肯斯坦引言

**何时使用：**
- 分析完成并经过验证后，用于框架设计和沟通
- 用于将洞察转化为演示文稿或报告
- 用于面向利益相关者的摘要（在别处验证后）
- **不用于主要分析**，如果可以避免的话

**最佳实践：**
- **在没有验证的情况下永远不要使用 ChatGPT 的引言**
- 用于综合，而不是发现
- 最适合包装，而不是分析
- 首先使用 Claude 或手动审查验证

## 并排示例

给定相同的面试记录：

| 方面 | Claude | Gemini | ChatGPT |
|------|--------|--------|---------|
| **发现的主题** | 8 个主题，全面覆盖 | 5 个主题，证据充分 | 6 个主题，遗漏价格敏感性 |
| **引言风格** | 逐字，更长的摘录 | 短片段，逐字 | 已总结，有时组合 |
| **置信度** | 保守评分 | 对较少主题的高置信度 | 对所有主题的高置信度 |
| **可靠性** | 高 - 保持基于数据 | 高 - 严格的证据链接 | **低** - 创意处理证据 |
| **最适合** | 主要分析 | 证据验证 | 利益相关者沟通 |

## 工作流程推荐

**使用多个模型的最佳工作流程：**

1. **发现**（Claude）- 运行综合分析以查找所有主题和模式
2. **证据验证**（Gemini）- 验证主题是否有充分证据
3. **引言验证** - 使用验证提示词（参见 [prompt-templates.md](references/prompt-templates.md)）
4. **框架设计**（ChatGPT）- 为利益相关者包装验证的洞察

**单模型工作流程（如果您只能访问一个）：**

- 只有 Claude？→ 您已经准备好了。使用验证提示词。
- 只有 Gemini？→ 期望提示 2-3 次以获得完整性。要求更长的引言。
- 只有 ChatGPT？→ 积极使用验证提示词。验证所有内容。

## 为什么结果不同

不同模型给出不同结果，因为：

1. **训练目标不同** - 一些针对有用性进行优化，其他针对准确性
2. **引言处理不同** - 一些检索，一些生成，一些总结
3. **置信度校准不同** - 一些过度自信，一些保守
4. **模式匹配与深度** - 一些优先考虑明显的模式，一些深入挖掘

这就是为什么验证很重要。大多数团队只有一个 LLM 企业账户，不会看到他们所选模型的缺点。

## 为您的情况选择合适的模型

**使用 Claude 当：**
- 您想要最好的主要分析
- 深度和细微差别很重要
- 您对验证证据感到满意
- 这是用于发现（而不仅仅是包装）

**使用 Gemini 当：**
- 您有视频面试
- 您更喜欢较少、证据充分的主题
- 您愿意多次提示
- 证据验证是您的优先事项

**使用 ChatGPT 当：**
- 分析已完成并验证
- 您需要为利益相关者包装洞察
- 您想要为演示文稿进行创意框架设计
- 您别无选择（验证所有内容！）

## 背景说明

这些建议基于的文章在示例中使用了 ChatGPT，因为：
- 它是产品团队中最广泛使用的模型
- 它最容易出现所涵盖的失败模式
- 修复在所有模型中都有效并改善结果

然而，Claude 被推荐用于分析工作，正是因为它不太容易受到这些失败模式的影响。

在以下情况下阅读本文档：
- 决定使用哪个 LLM 进行研究分析
- 理解为什么不同模型给出不同的结果
- 设计使用多个模型的工作流程
- 诊断 AI 分析质量为何变化
- 向利益相关者解释模型选择
