# 交互原型概念卡规范（Phase 4）

对用户选中的 1–2 个方向，输出完整概念卡，直接可用于 vibe coding。六个字段，每个都给好/坏例子。

> 贯穿原则：**语言风格必须和情绪内核的重量匹配**。沉的用沉的语言，轻的/荒诞的用轻的语言，绝不升华。

---

## 字段一：产品名 + 一句话定义

产品名不描述功能，**命名一种存在**。Claude 必须给出 **2–3 个候选**，不能留白或说「你随意命名」。

- ❌「全球日落直播工具」
- ✅「Eternal Sunset」
- ❌「专注计时器」
- ✅「专注飞机 / Flight」

一句话定义格式：**「一个让你___的___」**，填入具体感受 + 产品形态。
- ✅「一个让你在任何时刻都能找到一处日落的地方」

---

## 字段二：核心情绪时刻

用户打开产品的**第一秒**感受到什么。是感受描述，不是功能描述。

沉的：
- ❌「感到平静」「提高专注力」
- ✅「像突然想起来自己还活着」
- ✅「像有人轻轻拍了一下你的肩膀，没有说话」

轻的 / 荒诞的：
- ✅「打开之后嘴角压不住，但又不知道怎么跟别人解释这个东西」
- ✅「做了一件很蠢的事，但感觉良好」
- ✅「发现自己和一幅 300 年前的画像长得一样，然后哈哈哈哈哈」

---

## 字段三：交互隐喻

核心动作 + 它隐喻了什么。**必须是动词句**，格式：**「[用户动作] = [情绪意义]」**

- ✅「坚持专注 = 飞行距离」（专注飞机）
- ✅「等待日落 = 世界上有个地方和你一样」（Eternal Sunset）
- ✅「每两天按一次 = 向世界确认我还在」（死了么）
- ❌「用一种有趣的方式提升专注」（不是动词等式，太抽象）

---

## 字段四：视觉方向

一个强烈、具体的视觉意象，给设计师/vibe coding 的视觉输入。三要素必须齐全：

1. **色彩基调**：具体颜色，不是「暗色系」（✅「午夜蓝 + 暖金」）
2. **核心视觉动作**：不是静态描述（✅「字母像雨水从顶部落下」）
3. **情绪氛围**：可用比喻（✅「整个画面在呼吸」）

示例：
> 「午夜蓝的背景，白色字母像慢速雨水从顶部落下，速度和密度随用户当前情绪状态变化；落到底部时消失，像被地面吸收——整个画面在呼吸。」

---

## 字段五：Vibe Coding 提示词

直接可粘贴进 Cursor / Windsurf / Claude 开始构建原型。完整模板：

```
[产品名] 交互原型

视觉风格：[3–5 个英文关键词，用于 CSS/动画方向]
核心动效：[描述主要的动画和交互行为]
技术方向：[推荐实现方式，如 Three.js / Canvas / CSS Animation / WebGL]
情绪氛围：[2–3 个英文情绪关键词]
核心交互逻辑：[1–3 句话描述主要用户交互流程]

请构建一个可在浏览器中运行的交互原型，
重点实现[核心动效]，
整体风格参考[视觉风格关键词]，
让用户感受到[核心情绪时刻]。
```

五要素（视觉风格 / 核心动效 / 技术方向 / 情绪氛围 / 核心交互逻辑）缺一不可。

---

## 字段六：传播钩子

一句让人看到就想转发的话。不是 slogan，是**能出现在评论区置顶**的那种句子。Claude 给 **2–3 个候选**。

**判断标准**：这句话如果出现在小红书评论区，能不能被 1000 人点赞？

- ❌「功能很强大，推荐使用」
- ✅「事实上，只要搜这个名字就行了」（Eternal Sunset 真实置顶评论）
- ✅「我去。你简直就是天才」（思花真实置顶评论触发源）
- ✅「每两天按一次『我还活着』」（死了么）

---

## 概念卡整体质量自检清单

- [ ] 产品名给了 2–3 个候选，命名一种存在而非功能
- [ ] 核心情绪时刻是感受描述，不是功能描述
- [ ] 交互隐喻是「动作 = 情绪意义」的动词等式
- [ ] 视觉方向含具体颜色 + 动态行为 + 情绪氛围三要素
- [ ] vibe coding 提示词五要素完整，可直接粘贴
- [ ] 传播钩子给了 2–3 候选，符合「1000 人点赞」标准
- [ ] **全卡语言风格和情绪内核重量一致**，轻的没被升华成沉的
- [ ] 读完整张卡有具体情绪反应，不只是信息堆叠
