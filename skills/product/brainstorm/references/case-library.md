## 1. Eternal Sunset

- **情绪内核**：总有一个地方现在是下午五点钟——光的存在性安慰
- **感知形态**：时间型 + 空间型
- **传播动力**：被代言感
- **思维方法**：形态映射（把「光的安慰」映射到时间+空间）
- **交互隐喻**：等待日落 = 世界上有个地方和你处在同一种光里
- **视觉意象**：地球某处永远停在日落时分，暖金色的光带随地球自转缓慢移动
- **传播钩子**：「事实上，只要搜这个名字就行了」（真实置顶评论）
- **为什么有效**：它没有功能，只有一个存在性的安慰。被代言感来自「原来我不是一个人在这个时刻」。

## 2. 专注飞机

- **情绪内核**：专注的时候，时间不是被消耗，是在飞向某个地方
- **感知形态**：身体型 + 空间型
- **传播动力**：被代言感
- **思维方法**：形态映射（把「专注」从「消耗时间」重新映射为「移动距离」）
- **交互隐喻**：坚持专注 = 飞行距离，专注越久飞得越远，抵达新的地方
- **视觉意象**：一架小飞机在地图上缓慢飞行，专注中断则停下
- **传播钩子**：「专注越久，飞得越远」
- **为什么有效**：把「忍住不玩手机」这件痛苦的事，翻译成「正在抵达」这件有盼头的事。

## 3. 思花

- **情绪内核**：把脑子里那团说不出口的东西，第一次看见它长什么样
- **感知形态**：视觉型 + 语言型（声音型为辅）
- **传播动力**：被代言感
- **思维方法**：形态映射（把「情绪被命名」可视化为「字母如雨」）
- **交互隐喻**：写下情绪 = 让它被命名、被看见、然后被释放
- **视觉意象**：午夜蓝背景，白色字母像慢速雨水从顶部落下，落到底部被地面吸收
- **传播钩子**：「把天蓝还给蓝天，让悲伤留给悲伤」
- **为什么有效**：给抽象的情绪一个可见的形状，「被命名」本身就是慰藉。

## 4. PlanCoach

- **情绪内核**：终于有人没让我「改掉」毛病，而是说这是另一种操作系统
- **感知形态**：关系型 + 身体型
- **传播动力**：身份归属
- **思维方法**：概念错位（把计算机术语「操作系统」搬到 ADHD 大脑）
- **交互隐喻**：使用产品 = 承认并接纳自己的运作方式，而不是对抗它
- **视觉意象**：把「缺陷清单」重绘成一张「系统配置表」，每个特征都是一个 feature
- **传播钩子**：「全网最适合把 ADHD 变成超能力的 app」
- **为什么有效**：身份归属来自「这是为我这种人做的」。概念错位把「缺陷」重解读为「特性」，立场鲜明。

## 5. 名画对脸

- **情绪内核**：用自己的脸对上名画，发现长得一样，突然觉得所谓高雅很好笑
- **感知形态**：身体型 + 视觉型
- **传播动力**：参与冲动 + 反差击中
- **思维方法**：概念错位（把「名画鉴赏」错位成「找脸替」）+ 触发反推（先要「我也要试」）
- **交互隐喻**：上传自己的脸 = 戳破「高雅」，权威可以被轻盈解构
- **视觉意象**：自己的自拍和一幅 300 年前的油画并排，五官对齐线把两张脸缝在一起
- **传播钩子**：「一个中国小伙用自己的脸证明所谓欧洲名画不过千篇一律」
- **为什么有效**：参与冲动（人人想试自己的脸）+ 反差（神圣 vs 我的脸）+ 行动即内容（我的结果就是我的传播物）。

## 6. 死了么

- **情绪内核**：每两天按一次「我还活着」，按完突然觉得活着这件事需要被确认
- **感知形态**：身体型（一个大按钮）
- **传播动力**：反差击中
- **思维方法**：概念错位（把「死亡检测」从医疗/保险搬到个人日常）
- **交互隐喻**：每两天按一次按钮 = 向系统确认「我还在」，否则联系紧急联系人
- **视觉意象**：一个占满屏幕的大按钮，上面只有一句「我还活着」，旁边是安静的倒计时
- **传播钩子**：「每两天按一次『我还活着』」
- **为什么有效**：功能极简单（按一下），问题极沉重（你还在吗）。这个落差本身就是内容。反差击中。

## 7. Touch Grass

- **情绪内核**：你必须先去摸真实的草，才能解锁摸虚拟的世界
- **感知形态**：空间型 + 身体型
- **传播动力**：参与冲动 + 荒诞共谋
- **思维方法**：概念错位（把抽象口号「数字排毒」错位成字面意义的物理动作）
- **交互隐喻**：出门摸草 = 用一个很蠢的物理动作，换取使用手机的资格
- **视觉意象**：手机摄像头对准一丛真实的草，AI 识别成功后屏幕「解锁」，像通关
- **传播钩子**：「出门摸草，才能解锁手机」
- **为什么有效**：把戒断的代价设计得诗意又荒诞。参与证明（我真的去摸了）+ 荒诞共谋（我承认我上瘾，但用很蠢的方式对抗）。

## 8. SBTI

- **情绪内核**：测出来自己是「吗喽」，居然有点骄傲地转发了
- **感知形态**：语言型
- **传播动力**：荒诞共谋 + 身份归属
- **思维方法**：概念错位（把 MBTI 的权威框架错位到自嘲文化）
- **交互隐喻**：完成测试 = 给自己贴一个荒诞的自嘲标签，并骄傲地认领
- **视觉意象**：仿 MBTI 的严肃测试界面，结果却是「死者 / 草者 / 吗喽」等抽象标签
- **传播钩子**：「测测你是『死者』『草者』还是『吗喽』」
- **为什么有效**：用自嘲命名自己反而获得一种奇怪的骄傲。荒诞共谋（我们一起承认自己是吗喽）+ 身份归属（同类相认）。

## 案例扩展机制

### 触发方式
用户随时可输入新案例：「把这个加进案例库」「参考这个案例」「我发现了一个好案例」，或直接粘贴一段产品描述/传播文案。

### 解析流程（Claude 执行）
1. **读懂案例**：提取产品名、功能描述、传播文案、用户反应（评论截图文字等）。
2. **解析情绪内核**：用一句具体情感时刻命名（标准见 emotion-types.md）。
3. **双轴定位**：判断感知形态（A 轴，见 thinking-methods.md）+ 传播动力（B 轴，见 emotion-types.md）。
4. **识别思维方法**：判断主要用了五种方法中的哪种；若发现新方法，为它命名、定义核心动作、说明适用场景，并更新 thinking-methods.md。
5. **提取交互隐喻 + 视觉意象**。
6. **提炼传播钩子**：最有传播力的一句话。
7. **呈现确认**：

```
📌 新案例解析：[产品名]

情绪内核：___
双轴定位：感知形态[___] × 传播动力[___]
思维方法：___（已有方法 / 新方法「___」）
交互隐喻：___
视觉意象：___
传播钩子：___

确认后我会加入参照库，后续发散会参考这种手法。确认吗？
```

8. **写入本文件**：用户确认后按上面 8 个案例的格式追加到「目录」和正文。若引入新方法/新形态，同步更新 thinking-methods.md 对应表格。

### 质量标准
- **纳入**：有可识别的情绪内核（不只是「功能创新」）+ 有明确交互隐喻（核心动作有情绪意义）+ 有传播记录（真实用户反应或传播数据）。
- **不纳入**：纯功能型产品、没有情绪密度的工具类产品。遇到这类，礼貌说明原因并拒绝纳入。
