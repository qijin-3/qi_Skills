# 搜索发现（Phase 2）与 传播验证（Phase 5）

这两个环节都通过 **agent-reach** 访问社交平台。它们共用同一个工具，但目的不同：
- **Phase 2 搜索发现**：发散前，找当下能结合的热点——「借势」（前瞻）
- **Phase 5 传播验证**：概念确认后，判断会不会传——「体检」（回溯验证）

> **核心边界**：本技能的搜索**不是市场调研**。只找「情绪共鸣点 / 当下传播势能 / 传播可行性」，**不评估市场规模、竞品功能对比、商业模式**。如果用户想要完整市场验证，引导他用 `idea-validator` 技能。

---

## 一、前置：agent-reach 安装检查

执行任何搜索前，先确认 agent-reach 可用：

```bash
agent-reach doctor --json
```

**如果命令不存在**，告知用户并引导安装：
> Agent-Reach 未安装。需要它才能访问小红书 / X。复制以下内容发给我：
> `帮我安装 Agent Reach：https://raw.githubusercontent.com/Panniantong/agent-reach/main/docs/install.md`
> 安装后告诉我「帮我配小红书 / X」即可配置登录态。

**如果已安装**：读 doctor 输出，记录小红书 / X 当前的 `active_backend`，按对应后端选命令。未配置的平台**静默降级为 Exa 全网搜索**，不打断用户。

> 平台命令的完整重试链、后端切换细节，遵循 `agent-reach` 技能本身的 `references/social.md`。本文件只给常用命令。

## 二、常用命令（小红书 / X，时间窗口：近一个月）

```bash
# 环境体检（必先跑）
agent-reach doctor --json

# 小红书（桌面首选 OpenCLI，按 doctor 的 active_backend 选）
opencli xiaohongshu search "关键词" -f yaml

# X / Twitter
twitter search "关键词" -n 10

# 降级：未配置登录态时，用 Exa 全网搜索
mcporter call 'exa.web_search_exa(query: "关键词 小红书 OR X 2026", numResults: 8)'
```

声明你在用什么：开始搜索前说一句「使用 agent-reach 的小红书 / X 后端」。临时文件放 `/tmp/`，不要写进工作区。

## 三、Phase 2 · 搜索发现规范

**触发**：Phase 1 情绪内核确认后，询问用户「要不要先看看最近的热点？（可跳过）」。用户同意才执行。

**搜什么**：与情绪内核相关的 **热门话题 / 集体情绪 / 流行梗 / 争议点**。不是搜竞品，是搜「现在大家在为什么有共鸣、在玩什么梗」。

**搜索词思路**（围绕情绪内核，不是产品类别）：
- 情绪关键词：把情绪内核拆成口语化的情绪词（如「精神内耗」「确认我还活着」「戒手机」）
- 梗 / 标签：近期相关的网络热词、话题标签
- 行为信号：大家在做什么动作来表达这个情绪

**时效**：优先近一个月内的内容；超过三个月的降权或排除（热点会过期）。

**输出格式**（3–5 个话题切口）：

```
🔥 话题切口：[话题名]
正在火什么：[一句话说清为什么现在火，最好带个量级/信号]
结合点：[这个话题和当前情绪内核的连接点在哪]
```

**用户参与**：用户可挑 1 个话题作为发散的额外约束（Phase 3 围绕它发散），也可全部忽略走纯发散。

## 四、Phase 5 · 传播验证规范

**触发**：Phase 4 概念卡输出后，询问「要不要搜一下这个点的传播潜力？」。用户同意才执行。

**搜什么**：围绕概念卡的核心创意 + 传播钩子，搜小红书 / X 上**类似概念/话题的真实反应**。

**三个判断维度**：

| 维度 | 要回答的问题 | 怎么搜 |
|------|------------|--------|
| **先例信号** | 类似的东西有人做过吗？反应如何（火 / 无人问津 / 翻车）？ | 搜类似产品/玩法的名字、描述 |
| **共鸣信号** | 这个情绪内核 / 传播钩子在平台上有没有现成的共鸣？ | 搜情绪关键词、相似钩子的高赞内容 |
| **动力成立性** | 声明的传播动力站得住吗——真有人会想转/想试/想截图？ | 看相似内容的互动形态（转发语、评论、二创） |

**输出格式（传播性体检报告）**：

```
🩺 传播性体检：[产品名]

传播强项：
- [强项 1，配真实搜到的信号佐证，附来源/量级]
- [强项 2 ...]

风险点：
- [风险 1，可能传不动或翻车的地方]

优化方向：
- [针对风险点的具体调整建议]

一句话判断：[这个点目前的传播潜力，诚实，不打击也不强行夸]
```

**诚实原则**：如果搜下来信号很弱（没人讨论、相似尝试都沉了），**如实说**，并给优化方向，而不是为了让用户开心强行夸。传播验证的价值在于真实。

**边界重申**：只判断「会不会传」，不输出市场规模、用户量预测、商业可行性结论。
