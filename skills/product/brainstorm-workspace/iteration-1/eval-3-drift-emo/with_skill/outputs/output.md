▶ **Phase 1 · 情绪解码**

**1a · 先说感受**

这个 idea 给我的第一感觉是——**一个人深夜把房间灯全关了，只留手机屏幕那一小块光，有很多话想打出来，但把输入法的音量、把所有提示音、把存在感全调到了零。** 有点像「想被听见，但只敢用最小的声音说，而且最好说完没人发现是我」的那种感觉。

它最戳我的不是「emo」本身——emo 满大街都是——而是你加的那半句：**「又不好意思被人看见」**。这半句把它从「树洞 / emo 日记」那一类东西里拽了出来。真正的张力不是「我很难过」，是**「我同时想被看见，又怕被看见」**——手指悬在发送键上那几秒。

我先不急着发散。先跟你确认一下我抓的方向，再问你两个小问题（不一次问完）。

**1b · 两个定向问题**（单轮只问 2 个）

1. **目标情绪**：你希望深夜刷到这个东西的人，是「沉默地觉得『这就是我』」，还是「在不好意思里偷偷笑一下（带点自嘲那种）」？这决定它整体是偏克制的暖，还是带点荒诞的轻。
2. **态度立场**：这个 idea 在对抗什么？是在反对「难过就要说出来 / 要被治愈 / 要被人安慰」这套「情绪必须被处理掉」的现状吗？

> （假设用户确认：偏「沉默地觉得『这就是我』」为主，但允许带一点点自嘲的轻——因为「不好意思」本身就带着一丝羞赧的好笑，不是纯悲；立场上确实在对抗「情绪一定要被倾诉、被回应、被治愈」这套叙事，它想守住「我就是想自己消化，但又不想完全没人知道我来过」这块灰色地带。）

**1c · 命名情绪内核并确认**

```
情绪内核：深夜情绪涌上来，话打了一半，手指悬在发送键上，怕被人看见，最后自己删了——但又有点希望，刚好有人知道我今晚来过。
```

「我把这个 idea 的情绪内核理解为上面这句。准确吗？尤其是结尾那个『又有点希望有人知道我来过』——我觉得这才是你这个 idea 的引擎，纯粹的『删掉』太干净了，反而没张力。」

> （假设用户确认：准确，结尾那半句保留，这就是核。）

**克制气质守则（贯穿全流程）**：这是一个**中等重量、带羞赧边的**情绪。不升华成「关于现代人孤独的宏大叩问」，也不拉成直白宣泄（「把你的痛喊出来！」）。语言要像它本人一样——小声、半截、不好意思。

▶ **Phase 3 · 情绪地图发散**

漂移模式说明：下面 7 个方向，**4 个自由漂移 + 3 个概念错位**，全程不引用案例库里的任何现成案例，也刻意避开「emo 日记 / 树洞 / 匿名社交广场」这三种一看就像别的 app 的范式。每个方向标注思维方法 + 双轴定位。

核心设计支点：**「想被看见 ⊕ 怕被看见」这对矛盾，必须在交互机制里被解掉，而不是用文案喊出来。**

---

**【方向 1 · 只剩温度，读不到字】**
思维方法：**自由漂移（反转法）**——别的产品都让你「把话说清楚、被读懂」，我整个反过来：让你**被看见「有多重」，但永远没人能读到你写了什么**。
双轴定位：感知形态[视觉型] × 传播动力[被代言感]
交互隐喻：你打字 emo，文字在离开你手指的瞬间**实时碎成只剩情绪强度的光/热**——别人（或第二天的你）只能看到「此刻有人很重」，读不到一个字。
视觉意象：黑屏上，你打的每个字刚出现就晕开、糊成一团有温度的暖红光斑，越用力打、删得越多，那团光越烫、越抖；停手后慢慢冷却成灰。
情绪时刻：「我把想说的全打出来了，它确实被发出去了、被看见了——但没人知道我说了什么。刚好。」

**【方向 2 · 深夜玻璃】**
思维方法：**自由漂移（极端化：最公开 × 最匿名同时拉满）**——把「怕被看见」推到极端私密、把「想被看见」推到极端公开，让两个极端在同一块屏幕上同时成立。
双轴定位：感知形态[空间型] × 传播动力[身份归属 / 荒诞共谋]
交互隐喻：一面**只在深夜（比如 23:00–5:00）才亮起来的集体玻璃**，所有此刻在线的人的情绪变成一格格模糊光斑浮在同一块黑玻璃上；你点亮属于你的一格，**没有任何人能知道哪一格是你**，包括你自己第二天回来也认不出。
视觉意象：一整面深蓝近黑的雾面玻璃，上面飘着几十上百颗呼吸般明灭的小光点，像很远的城市夜窗，谁也看不清谁，但你知道「现在醒着 emo 的不止我一个」。
情绪时刻：「我没说话，也没人认识我，但我点亮的那一格，混在所有人的光里。我们都没睡，挺好。」

**【方向 3 · 退潮线】**
思维方法：**自由漂移（隐喻跳跃：emo → 水 → 涨潮 → 退潮后沙滩上的痕迹）**——情绪不是要被处理掉的东西，是会自己涨自己退的潮。
双轴定位：感知形态[时间型 + 身体型] × 传播动力[被代言感]
交互隐喻：emo 上来时你**什么都不用写，只要按住屏幕「站在那里」陪它涨**；松手即退潮，潮水退去后在沙上留下一条**只有你能看见的水痕**——那是你今晚来过的唯一证据，第二天自动被新的潮抹平。
视觉意象：深夜色的海面缓缓涨到屏幕中央，你按住时浪一点点变高变响，松手后哗地退下去，湿沙上留下一道弯弯的盐白痕迹，几秒后被下一道浪舔掉。
情绪时刻：「我没有解决它，我只是按住屏幕陪它涨了一会儿，然后看它自己退了。原来我不用做什么。」

**【方向 4 · 情绪小票】**
思维方法：**自由漂移（随机词碰撞：emo × 收据/购物小票）**——抓一个和情绪八竿子打不着的物件「小票」，强行和深夜情绪焊在一起，看迸出什么。
双轴定位：感知形态[语言型 + 视觉型] × 传播动力[反差击中]
交互隐喻：你每一次深夜 emo，app 给你**打印一张「情绪小票」**——时间、强度、本次「想发送但撤回」次数、「打了又删的字数」，像一张你深夜情绪的购物清单，**没消费成功，但留了底**；攒久了你有一叠卷边的小票。
视觉意象：黑底上一张窄窄的热敏纸小票，齿孔、虚线、模糊的等宽字体，底部一行小字「本次未送达 ×3 / 谢谢光临」，纸边微微卷。
情绪时刻：「我今晚什么都没发出去，但手里多了一张小票，证明我来过、犹豫过、又算了。荒唐，但有点想留着。」

**【方向 5 · 情绪黑匣子】**
思维方法：**概念错位**——把【黑匣子 / 航空事故记录仪】，从【航空安全·事故调查】语境，搬到【个人深夜情绪】语境。原概念里「黑匣子平时锁死、只有坠机才被读取」这一点，正好对上「想被看见但只在最极端时」。
双轴定位：感知形态[空间型 + 语言型] × 传播动力[被代言感]
交互隐喻：深夜的话录进**一个平时打不开的黑匣子**，**只有「失事」那天才会被读取**——你某天真的撑不住，主动按下「请求读取」，它才把你历次深夜留言交给一个你**预先指定的人 / 未来的自己**。
视觉意象：一个橙色（航空黑匣子真实是橙色）的金属盒沉在屏幕底部黑色里，平时只有一点冷光呼吸，盒身写着「FDR · 仅在坠毁后读取」，按一次只是确认录入，盒盖不开。
情绪时刻：「我把今晚锁进去了。它在那。等到我真的不行那天，会有人读到——但今晚不会，今晚只有我和这个盒子。」

**【方向 6 · 暗房】**
思维方法：**概念错位**——把【暗房显影 / 胶片摄影冲洗】，从【摄影·化学冲洗】语境，搬到【深夜情绪】语境。借「显影必须在黑暗里泡、见光即定影/过曝」这套物理，做成情绪的延迟与自毁。
双轴定位：感知形态[时间型 + 视觉型] × 传播动力[被代言感]
交互隐喻：你深夜写下的话**不会立刻出现**，必须「泡在黑暗里一整夜」慢慢显影；**天亮才显影完成，但只停留几分钟就过曝消失**——你被迫在还没人醒来的清晨，独自看一眼昨晚那个自己。
视觉意象：深红暗房灯下，一张空白相纸浸在显影盘里，文字像影像一样从无到有缓缓浮起来；天光一进来，相纸迅速发白、字迹被光吃掉。
情绪时刻：「我昨晚写的东西，今早自己浮出来了，就给我看了三分钟。看完它就白了。像我跟昨晚的我，匆匆碰了个面。」

**【方向 7 · 深夜情绪急诊】**
思维方法：**概念错位**——把【急诊分诊台 / 夜间挂号候诊】，从【医院·医疗急救】语境，搬到【深夜情绪】语境。借「分诊、挂号、候诊区、永远叫不到号」这套流程的荒诞感，给「不好意思被看见」留一点自嘲的轻。
双轴定位：感知形态[空间型 + 声音型] × 传播动力[荒诞共谋 / 身份归属]
交互隐喻：你「挂号」描述今晚的症状，系统给你分一个**分诊级别**（比如「III 级·可自愈但建议留观」），然后让你**在候诊区等待**——号永远不会被叫到，但候诊区里飘着**其他匿名号码的模糊呼吸声 / 翻动杂志的窸窣声**，陪你到天亮，天亮自动「离院」，不留病历。
视觉意象：深夜医院走廊那种惨白偏冷的灯，一块叫号屏永远停在「正在为您安排…」，候诊椅上坐着几个半透明的模糊人影，谁也不看谁，远处有规律的滴答声。
情绪时刻：「我挂了个深夜情绪的号，分诊说我『可自愈』。号一直没叫到，但旁边一直有人在呼吸。挺好，我也不是真想被叫到。」

**质量自检（已逐项过）**：
- 7 个方向均为漂移模式产物（4 自由漂移 + 3 概念错位），**0 个案例库现成案例的变体**，刻意避开 emo 日记 / 树洞 / 匿名广场三种常见范式。
- 每个方向都解了「想被看见 ⊕ 怕被看见」这对矛盾的**机制**（读不到字 / 认不出是谁 / 不留痕 / 没送达 / 锁死到失事 / 过曝消失 / 叫不到号），不是靠文案喊。
- 全程克制：保留羞赧、半截、小声的气质，没有升华成宏大叩问，也没拉成直白宣泄；方向 4、7 留了一点自嘲的轻，对应「不好意思」里那一丝好笑。

**「哪 1–2 个方向最让你有感觉？」**
> （假设用户确认：最有感觉的是 **方向 1「只剩温度，读不到字」**——因为它把那对矛盾解得最干净、最不像任何现成 app；想搭配看 **方向 6「暗房」** 的延迟自毁机制。）
>
> 我的推荐也一致：**主推方向 1**。理由——它的传播动力（被代言感）最稳，而且「被看见强度、但读不到内容」这一个机制本身就是一句能让人「啊这不就是我」的话，最适合往下做成概念卡。

▶ **Phase 4 · 交互原型概念卡**（方向 1 · 只剩温度，读不到字）

**① 产品名 + 一句话定义**
候选名（命名一种存在，不描述功能）：
- **余温 / Warmth**
- **糊 / Blur**
- **噪 / Static**

一句话定义：**「一个让你被人看见『此刻有多重』，却没人能读到你写了什么的地方。」**

**② 核心情绪时刻**
打开第一秒：
> 「我把想说的全打出来了，越打越用力，它确实被发出去、被看见了——可没有一个人知道我说了什么。刚好。」

**③ 交互隐喻**
> **打字越用力、删得越多 = 那团光越烫、越抖（被看见的是温度，不是内容）。**

**④ 视觉方向**
- **色彩基调**：近黑的深夜蓝底 + 一团会呼吸的暖红 / 余烬橙（情绪光斑），冷却后转灰白。
- **核心视觉动作**：每个字刚被打出就立刻晕开、糊成一团光，**永不成形为可读文字**；打字频率/删除次数实时驱动光团的亮度、温度色相和抖动幅度；停手后光团缓慢冷却、变灰、下沉消失。
- **情绪氛围**：像隔着一层磨砂玻璃看一炉将熄未熄的火——**你知道里面有话，但永远读不到**，整个画面在轻轻发烫又慢慢凉下去。

**⑤ Vibe Coding 提示词**（可直接粘贴）
```
余温 / Warmth 交互原型

视觉风格：deep-night blue-black、ember glow、frosted blur、soft bloom、grainy noise
核心动效：用户在隐藏输入框打字，每个字符在输入瞬间不渲染为文字，而是在画布对应位置生成一团高斯模糊的暖色光斑（warm red→ember orange）；打字速度、删除（backspace）次数实时映射为光斑的亮度、色温与抖动/扩散半径——越激烈越烫越抖；停止输入 3 秒后光斑缓慢冷却（暖→灰）、下沉、淡出。永远不显示任何可读文字。
技术方向：HTML5 Canvas / WebGL（shader 做 bloom + blur 最佳），或 Canvas2D + CSS filter: blur() 快速实现；输入用隐藏 textarea 捕获事件，绝不回显文本。
情绪氛围：intimate、restrained、seen-but-unreadable
核心交互逻辑：用户深夜打字倾诉 → 文字实时碎成只剩情绪强度的光，自己和他人都读不到内容，只能感到「此刻有人很重」→ 停手后光自动冷却消失，不留可读记录。

请构建一个可在浏览器中运行的交互原型，
重点实现「文字输入即碎成暖色模糊光斑、强度由打字/删除节奏驱动、停手后冷却消失」的核心动效，
整体风格参考 deep-night blue-black + ember glow + frosted blur，
让用户感受到「我全说出来了、也被看见了，但没人读得到一个字，刚好」。
```

**⑥ 传播钩子**（能在评论区被 1000 人点赞）
- **「它能看见我有多重，但读不到我写了什么。」**
- **「深夜想说话，又怕被看见——这个刚好，说了等于没说，但又不是真的没说。」**
- **「我打得越用力，它越烫，可没人知道我说了啥。我需要的就是这个。」**
