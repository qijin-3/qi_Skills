# 任务识别 · 标题与描述

从用户原话中提取任务，必要时轻度润色，**标题与描述分工明确**。

## 标题

- **尽可能简洁**：动词短语，≤30 字
- 去掉语气词、重复、背景铺垫，只留「做什么 + 对什么」
- 保留用户原意，不过度改写

## 描述

- 标题装不下的**补充信息**写入描述：背景、原因、具体步骤、约束、参考链接
- 对用户原话做**轻度润色**：修正错别字、理顺语序，不增删事实
- 用户只说了一短句、无补充 → **不设描述**（创建时省略 `--description`）

## 判断规则

| 用户输入 | 处理方式 |
|----------|----------|
| 短句，意图清晰 | 整句压缩为标题，无描述 |
| 长段 / 多条细节 / 有背景说明 | 提炼简洁标题 + 其余润色后写入描述 |
| 多条可执行事项 | 拆成多个任务，各自独立标题/描述 |

## 示例

**短句**

| 用户说 | 标题 | 描述 |
|--------|------|------|
| 良渚租房 | 良渚租房 | — |

**长内容**

| 用户说 | 标题 | 描述 |
|--------|------|------|
| 我想写一篇关于 HTML 为什么是最好的 AI 沟通媒介的文章，主要论证 canvas、artifact 相比 markdown 的优势，还可以讲讲交互性 | 写 HTML 是最佳 AI 沟通媒介的文章 | 论证 canvas、artifact 相比 markdown 的优势，以及 HTML 的交互性 |

**带约束**

| 用户说 | 标题 | 描述 |
|--------|------|------|
| 更新小红书店铺类目资质，上次被驳回了，说是经营范围和营业执照不一致，得先改执照再重新提交 | 更新小红书店铺类目资质 | 上次驳回原因：经营范围与营业执照不一致。需先改执照再重新提交。 |

## 截止日期

仅用户**明确提到**时提取（如「明天」「下周五」「6/30 前」），否则不设。
