# Health Daily Remind（每日健康提醒）

## 触发

- **手动**：用户说「今日健康计划」「健康提醒」

## ⚠️ 只读！不做任何写入

不写入 Base、不修改 Calendar、不创建任何文件。

## ⚠️ 计划已提前排好，只呈现不生成

**绝对禁止**：重新生成、推断或修改今日训练计划及时段。

今日训练的时间与内容来自 **`state/current-week.json`** 中当日的 `sessions` 数组（由 `health-weekly-plan` 周一排定）。按 `sessions[].time` 顺序原样呈现，**不使用**固定时间锚点（如 15:00 / 19:30）去猜测或过滤。

## 工作流程

```
1. 读取本地上下文
   content/profile.md（长期习惯，非本月目标）
   content/{YYYY}/{MM}/monthly-plan.md（本月饮食/运动侧重点）
   state/current-week.json → 取今日 sessions（time / label / content）

2. 读取近期数据摘要（本地脚本）
   python3 health-system/scripts/health_analytics.py --daily-remind

3. 生成前情摘要 + 今日时间线
   时间线严格按 sessions[].time 排序，内容用 sessions[].content

4. 飞书发送（≤10 行）
```

## 消息格式（≤10 行）

### 多场次日（示例，时刻以 JSON 为准）

```
📋 周四 · 上肢力量

昨天没运动，今天晚间稳扎稳打。

15:00 🧘 微训 → 靠墙静蹲 2×30s，腿髋拉伸
19:30 💪 上肢 → 跳绳 300，俯卧撑 4×15，仰卧起坐 4×20

🍳 早餐：水煮蛋2个+无糖豆浆1杯+全麦面包1片
🥗 午餐：糙米饭150g+清蒸鱼200g+西兰花200g
🍚 晚餐：小米粥1碗+凉拌黄瓜+鸡胸肉150g

有变化直接说。
```

### 单场次日

```
📋 周三 · 下肢力量

护膝：膝盖不超脚尖、不内扣。

19:30 🦵 下肢 → 自重深蹲 4×15，臀桥 3×15

🍳 早餐：牛奶1杯+燕麦片50g+香蕉1根
🥗 午餐：杂粮饭100g+虾仁150g+青菜200g
🍚 晚餐：蒸蛋羹+紫菜蛋花汤+清炒豆角
```

### 恢复日

```
📋 周日 · 恢复日

好好休息。

21:00 🧘 散步+泡沫轴
🍳 全天轻食
```

> 上方时刻仅为示例；实际以 `current-week.json` 当日 `sessions` 为准。

## 前情摘要规则

| 场景 | 怎么说 |
|------|--------|
| 连续完成 | 「连续 N 天完成，保持节奏。」 |
| 昨日未运动 | 「昨天休息了，今天晚间完成。」 |
| 体重上升 | 「体重连涨，今天晚餐控碳水。」 |
| 饮食偏差 | 「昨天高升糖偏多，今天早餐换蛋白质。」 |

## 饮食建议原则

- ✅ 每餐具体食物（水煮蛋2个，不是「高蛋白」）
- ✅ 包含份量（牛奶1杯，糙米饭150g）
- ✅ 符合当月 monthly-plan 饮食重点
- ❌ 标签式建议（「蛋白+蔬菜」→ 换成「水煮蛋2个+西兰花200g」）

## 硬规则

- 不写 Base，不改 Calendar
- 不读飞书 Calendar / 飞书文档做计划回源
- 时段只来自 `current-week.json`，禁止硬编码锚点
- ≤10 行，先结论再论据

`references/config.md`（操作指引）。Token/字段 ID 由脚本读取 `system.json`，AI 不碰。
