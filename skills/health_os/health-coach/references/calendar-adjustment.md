# 日历调整规则

用户明确要**改今天/明天的训练安排**时读本文件。单纯记录「练了什么」不写 Calendar。

## 计划真相源

本周已排计划以 `state/current-week.json` 为准；Calendar 是用户视图。调整训练时须**同时**更新 Calendar 事件与 `current-week.json` 中对应日期的 `sessions`，避免提醒与实际不一致。

## 时段唯一性

每个时段最多一个运动事件：

| 时段 | 时间范围 |
|------|---------|
| 早上 | 06:00–12:00 |
| 下午 | 12:00–18:00 |
| 晚上 | 18:00–23:59 |

新增或推迟前，先查目标日该时段是否已有运动事件。若已有：告知用户「该时段已有 [事件名]，替换还是合并？」，等确认再操作。

## 操作权限

| 操作 | 条件 |
|------|------|
| 删除事件 | 用户明确说「删除/取消/不练了」 |
| 新增事件 | 用户明确说「加一场/增加运动/补个训练」 |
| 改内容或时间 | 情境变更时 **update** 已有事件，不删了重建 |

## 查询今日 Calendar（仅调整时用）

```bash
lark-cli calendar +agenda \
  --start YYYY-MM-DD --end YYYY-MM-DD+1 \
  --as user --format json
```
