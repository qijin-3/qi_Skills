# daily-log.md 格式规范

health-coach（白天追加）与 health-evening-close（晚间定稿 + 同步 Base）共用本文件。

**路径**：`~/SynologyDrive/Sync/OS/Health_OS/content/{YYYY}/{MM}/daily-log.md`

## 写入原则

| 时机 | 谁写 | 写哪里 |
|------|------|--------|
| 白天用户口述 | health-coach | 仅 daily-log.md（**不写 Base**） |
| 每晚 21:30 | health-evening-close | 读 daily-log → 同步 Base → 定稿当日段落 |

同一天只有一段 `## YYYY-MM-DD`。coach 首次记录时创建该段，后续**就地更新**对应子节，不另起第二段。

## 白天段落模板（coach 维护）

```markdown
## YYYY-MM-DD

**体重**: Xkg（或未测）
**体脂**: X%（或未测 / 沿用 YYYY-MM-DD）
**睡眠**: 就寝 YYYY-MM-DD HH:MM · 起床 YYYY-MM-DD HH:MM · 共 X.Xh · 异常醒来 X.Xh（或无）

**饮食**
- 早餐：食物(份量), …（约 XXXkcal）
- 午餐：…（约 XXXkcal）
- 晚餐：…（约 XXXkcal）
- 加餐：…（约 XXXkcal）（若有）

**运动**
- HH:MM | 类型 | 内容 | 时长Xmin
（无运动则写：无）

**身体状态**: 标签1, 标签2（无则写：无）
**当日情境**: 标签1, 标签2（无则写：无）
**备注**: 不适合进结构化字段的口述（无则省略本节）
```

### 各节更新规则

**体重 / 体脂**：用户提及即更新对应行。

**睡眠**（有则必记）：
- 归属**就寝那天**的日期段（见 `recording-rules.md` 睡眠归属）
- 跨午夜示例：6/25 01:00 睡、07:00 起 → 写在 `## 2026-06-24` 的睡眠行，**不是** 6/25
- 格式：`就寝 YYYY-MM-DD HH:MM · 起床 YYYY-MM-DD HH:MM · 共 X.Xh`；有中途清醒则追加 `· 异常醒来 X.Xh`
- **共 X.Xh** = 各段实际入睡时长之和（不含清醒时段）
- **异常醒来 X.Xh** = 入睡期间中途清醒总时长（小时），多段累加；无则省略
- 用户早上报睡眠时，coach 打开**就寝日**对应段落写入，若该月文件不存在则创建

**分段睡眠示例**（就寝日 6/24，看世界杯）：

用户口述：0 点睡到 4 点 → 看球 2h → 6 点睡到 8 点 → 看球 2h → 再睡

```
**睡眠**: 就寝 2026-06-24 00:00 · 起床 2026-06-24 10:00 · 共 6.0h · 异常醒来 4.0h
```

- 实际睡眠：4h + 2h = 6h（若之后还有睡眠段落，继续累加）
- 异常醒来：2.0 + 2.0 = 4.0h
- 分段明细可写入 **备注**（如「00:00–04:00 睡；04:00–06:00 看球；06:00–08:00 睡；08:00–10:00 看球」）

**饮食**：有餐次即追加/更新对应行；卡路里由 coach 估算写入括号内。

**运动**：每场次一行，格式 `HH:MM | 类型 | 内容 | 时长Xmin`；时长来自用户口述或合理推断，**只存在运动文本中**，不单独猜「全天运动时长」字段。

**身体状态**（多选标签，从用户口述提取）：

| 标签 | 触发语示例 |
|------|-----------|
| 膝盖不适 | 膝盖疼、护膝、深蹲疼 |
| 腰背不适 | 腰疼、久坐酸 |
| 肩颈不适 | 脖子僵、肩膀酸 |
| 肠胃不适 | 拉肚子、胃胀 |
| 感冒/过敏 | 流鼻涕、过敏 |
| 疲劳 | 太累、没精神 |
| 睡眠不佳 | 失眠、睡得差（非具体时点） |
| 精力充沛 | 状态很好、精神足 |

**当日情境**（多选标签，从用户口述提取）：

| 标签 | 触发语示例 |
|------|-----------|
| 外食 | 在外面吃、食堂、外卖 |
| 聚餐/应酬 | 请客、饭局、喝酒 |
| 出差/旅行 | 出差、在外地 |
| 时间紧张 | 太忙、来不及做饭 |
| 自己做饭 | 在家做、带了饭 |
| 加班/熬夜 | 加班、熬夜、看球晚睡 |
| 休息日 | 今天休息、没安排 |
| 月经/生理期 | 生理期（用户提及） |

同天多次口述 → **合并**标签（去重），不要覆盖掉已有标签。

**备注**：自由文字；与身体状态/当日情境不重复的内容放这里（如「体脂沿用昨天」「外食估算」）。

## 晚间定稿格式（evening-close 输出）

晚间收口在同步 Base 后，将当日段落**重写**为定稿（结构同上，并补全衍生字段）：

```markdown
## YYYY-MM-DD

**体重**: Xkg | **睡眠**: 就寝 HH:MM · 起床 HH:MM · 共 X.Xh · 异常醒来 X.Xh

**饮食**
- 早餐：...（约 XXXkcal）
- 午餐：...（约 XXXkcal）
- 晚餐：...（约 XXXkcal）
- 加餐：...（约 XXXkcal）（若有）
- 合规度：优/良/一般/差

**运动**
- HH:MM | 类型 | 内容 | 时长Xmin
（无运动则写：无）
**训练强度**: 无/轻度/中等/较高
**运动完成率**: XX%（若有周计划排期）

**身体状态**: 标签1, 标签2
**当日情境**: 标签1, 标签2
**备注**: …
```

定稿时：
- **不写**「全日约 XXXXkcal」—— Base 中 `全日卡路里_kcal` 为公式字段，由表内公式自动汇总
- 睡眠行无数据且当天/就寝日用户均未提及 → 保留 `**睡眠**: 无` 或留空并在收口日志注明

## 文件操作

```bash
# 路径中的 YYYY、MM 按记录归属日期（睡眠按就寝日）
LOG=~/SynologyDrive/Sync/OS/Health_OS/content/YYYY/MM/daily-log.md
```

coach / evening-close 直接读写该文件；**禁止**为同日创建第二个日志文件。
