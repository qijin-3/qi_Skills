# 数据写入规则

记录饮食、运动、睡眠、体重前必读。

## 白天只写本地，晚间统一入 Base

| 时段 | 写入目标 | 方式 |
|------|---------|------|
| **白天**（health-coach） | `content/{YYYY}/{MM}/daily-log.md` | 直接编辑日志文件 |
| **晚间**（health-evening-close） | 飞书 Base | `health_base.py` upsert（从 daily-log 同步） |

白天 coach **禁止**调用 `health_base.py` 或 `lark-cli base`，减少飞书交互。用户口述的所有结构化数据先进入 daily-log，由晚间收口一次性落库。

格式见 [`daily-log-format.md`](daily-log-format.md)。

## 用户只说不填表

用户**从不**打开或填写飞书 Base，只用自然语言口述。**白天由 health-coach 写入 daily-log**；**晚间由 health-evening-close 同步到 Base**。

禁止要求用户「去 Base 填一下」；缺什么 coach 当场记入 daily-log，或由晚间收口从日志同步。

## 每天一条记录

每个日期在 Base 中只有一条记录（晚间 upsert）。脚本会先 `find_record_by_date` 再更新或创建。

## 睡眠归属

睡眠时长算在**上床那天**，不是起床那天。

**示例**（6月25日 01:00 睡、07:00 起）：

| 写入日期（daily-log 段落） | 就寝 | 起床 | 睡眠时长 |
|--------------------------|------|------|---------|
| **6月24日** | 2026-06-25 01:00 | 2026-06-25 07:00 | 6h |
| 6月25日 | — | — | （此次睡眠已在 24 日） |

用户早上说「昨晚睡了 6 小时」→ coach 写入**就寝日**段落的 `**睡眠**` 行，不是今天。

## 异常睡眠（分段清醒）

Base 字段 **`异常睡眠`**（数字，单位**小时**）记录一次夜间入睡过程中、因非睡眠目的中途清醒的总时长。晚间收口从 daily-log 同步。

| 概念 | 规则 |
|------|------|
| 计入 | 入睡后、最终起床前的清醒时段（看球、刷手机、起夜等） |
| 不计入 | 尚未入睡前的清醒；次日白天正常活动 |
| 多段 | 各段清醒**累加**为 `异常醒来 X.Xh` |
| 共 X.Xh | 仅各段**实际睡眠**之和，不含清醒时段 |

用户描述分段睡眠时，coach 应：
1. 解析各段睡/醒时间
2. 计算实际睡眠总时长 → `共 X.Xh`
3. 计算清醒总时长（小时）→ `异常醒来 X.Xh`
4. 复杂分段在 **备注** 留一行明细

用户只给总数（如「中间醒了 4 小时」）→ 直接写 `异常醒来 4.0h`，就寝/起床/共睡时长仍尽量补全。

## 卡路里（必填，写在 daily-log）

记录任何餐次时，**同步在 daily-log 餐次行写入估算卡路里**。不允许以「外食不清楚」留空；不确定时给保守估算，并在**备注**标「外食估算」或「聚餐估算」。

| 场景 | 精度 | 备注 |
|------|------|------|
| 自己做饭，食材明确 | ±50 kcal | — |
| 外食，菜品已知 | ±100 kcal | 外食 |
| 外食，菜品模糊 | 区间中值 | 外食估算 |
| 聚餐，难以估算 | 保守偏高 | 聚餐估算 |

常见食物参考 kcal → 读 [`calorie-reference.md`](calorie-reference.md)。

> `全日卡路里_kcal` 是 Base **公式字段**，禁止 AI 填写；晚间收口也不同步该字段。

## 文本格式

**餐次**（daily-log 与 Base 一致）：`食物(份量), 食物(份量); 标签`  
**运动**：`HH:MM | 类型 | 内容 | 时长min`

运动时长只写在运动文本末尾，**不设独立的「运动时长_分钟」字段**。

只写有值字段；未提及的字段不要写 null 占位。

## 备注

**备注**收录不适合进结构化字段的文字：身体不适、外食/聚餐/出差等情境、睡眠分段明细、体脂沿用、外食估算说明等。

- 同天多次口述 → **追加合并**（用 `；` 分隔，去重）
- 不再使用 `**身体状态**` / `**当日情境**` 标签行（已废弃，勿写入 Base）

## 训练强度

表示**当日实际运动**强度总结：`无` / `轻度` / `中等` / `较高`。

- 用户口述且能判断时 → coach 可在 daily-log 运动节后标注
- 未写时 → 晚间收口根据运动文本中的时长推断（见 health-evening-close）

## 日历调整

用户要改今天/明天训练安排时 → 读 [`calendar-adjustment.md`](calendar-adjustment.md)。日常记录不涉及 Calendar。
