# Health Evening Close（每日晚间收口）

## 触发

- **定时**：由调度系统每天 21:30 调用
- **静默执行**：不向用户发消息，技能完成自己的 task 即可

## Base 写入原则

用户**从不手填 Base**，只口述；白天由 **health-coach** 识别并写入，晚间由 **本技能** 补全仍为空的衍生字段。

飞书 Base **仅在**以下时机更新：
1. 用户口述 → health-coach 解析写入（含 `备注`、餐次、运动、睡眠、体重等）
2. **本技能晚间收口**（仅补空，不覆盖 coach 已写内容）

## 工作流程

```
1. 晚间完整收口（脚本一步完成）
   python3 health-system/scripts/health_analytics.py --evening-close
   或等价：--apply-evening

   脚本自动检查并补填当日仍为空的字段：
   - 餐次_卡路里（有餐次文本无卡路里）
   - 运动时长_分钟（有运动记录无时长）
   - 运动类型（有运动记录无类型）
   - 训练强度（当日运动强度总结，见下）
   - 运动完成率_%（对照 current-week.json）
   - 饮食合规度

2. 写入本地 daily-log.md
   路径：~/SynologyDrive/Sync/OS/Health_OS/content/YYYY/MM/daily-log.md
   追加今日完整日志（见格式）

3. 若 suggest_tomorrow_downgrade=true：
   在 daily-log 或本地 current-week.json 记提示；**不写明日 Base 训练强度**
```

## 字段语义（2026-07 起）

| 字段 | 含义 | 何时写 |
|------|------|--------|
| **训练强度** | 当日**实际运动**强度总结：`无` / `轻度` / `中等` / `较高` | coach 可写；未写时晚间补填 |
| **备注** | 用户口述的文字信息汇总（睡眠、体脂沿用、情境等） | **health-coach 从口述提取**；晚间仅当仍为空时不覆盖地跳过 |
| ~~训练主题~~ | 已删除 | 计划主题见 `current-week.json` |
| ~~饮食备注~~ | 已更名为 **备注** | — |

### 训练强度推断规则（脚本）

| 条件 | 值 |
|------|-----|
| 无运动记录 | 无 |
| 总时长 < 20min | 轻度 |
| 20–45min | 中等 |
| ≥45min 或含波比/HIIT 等 | 较高 |

## daily-log.md 追加格式

```markdown
---

## YYYY-MM-DD

**体重**: Xkg | **睡眠**: 就寝 HH:MM · 起床 HH:MM · 共 X.Xh

**饮食**
- 早餐：...（约 XXXkcal）
- 午餐：...（约 XXXkcal）
- 晚餐：...（约 XXXkcal）
- 加餐：...（约 XXXkcal）（若有）
- 全日约：XXXXkcal | 合规度：优/良/一般/差

**运动**
- HH:MM | 类型 | 内容 | 时长Xmin
（无运动则写：无）
**训练强度**: 无/轻度/中等/较高

**备注**: （当日文字记录汇总）
```

## 运动完成率规则

对照 `state/current-week.json` 当日 `sessions`：
- 有排期且已记录运动（运动_1/2/3 任一有值）→ 100%
- 有排期无记录 → 0%
- 恢复日无强制场次 → 不写入

## 写入字段汇总（仅补空）

| 字段 | 来源 |
|------|------|
| 早/午/晚/加餐_卡路里 | 脚本估算补填 |
| 运动时长_分钟 | 从运动文本解析或估算 |
| 运动类型 | 从运动文本推断 |
| 训练强度 | 当日运动强度总结 |
| 运动完成率_% | 对照 current-week.json |
| 饮食合规度 | 脚本规则推断 |

> 不向用户发消息。本技能只完成数据写入任务。
