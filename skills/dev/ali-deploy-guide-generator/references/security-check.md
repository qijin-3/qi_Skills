# 部署前敏感信息检查

在写入或修改部署相关文件之前执行，输出：**通过 / 需注意 / 需修复**。

## 1. 环境文件

- 确认 `.env`、`.env.local`、`.env.*` 在 `.gitignore`。
- 确认 `git ls-files` 未跟踪上述文件。若已跟踪：提示用户从 Git 移除并加入 .gitignore，文档中不写真实密钥。

## 2. 硬编码密钥

在源码与配置（*.ts、*.tsx、*.js、*.jsx、*.vue、*.json、*.yaml）中搜索，忽略注释与明显示例：

- `api[_-]?key\s*[:=]\s*['\"]?[a-zA-Z0-9_\-]{20,}`
- `secret\s*[:=]\s*['\"]?[a-zA-Z0-9_\-]{20,}`
- `password\s*[:=]\s*['\"]\S+['\"]`
- `token\s*[:=]\s*['\"]?[a-zA-Z0-9_\-]{20,}`
- `private[_-]?key`、`credential`、`ghp_`、`sk-`

若发现疑似硬编码：列出文件与行号，建议改为环境变量并在文档中说明在服务器/控制台配置。

## 3. 前端暴露

- **Vite**：仅 `import.meta.env.VITE_*` 打进前端包；提醒勿将 API Key/Secret 放入 VITE_*。
- **CRA**：仅 `REACT_APP_*` 暴露；同上。
- **Next.js**：仅 `NEXT_PUBLIC_*` 暴露；同上。

文档「环境变量」小节说明：敏感变量在服务器/控制台配置，不写入仓库。

## 输出

- 简短列表：通过 / 需注意 / 需修复。
- 若有需修复：先给出修改建议（如从 Git 移除 .env、改为 process.env.XXX），再继续生成方案。
